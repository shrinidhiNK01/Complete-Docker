Docker:- Contenarization Platform 
Package apps with all dependencies into isolated environments called Containers
Docker runs on any System without worrying about dependencies, OS Versions, or Conflicts
Faster deployment as containers start instantly compared to traditional machines(Virtual Machines)
Make sacling spps easy by running Multiple containers as Microservices


Without Docker:- It works in our machine , But not in urs 
              :- Apps break when moved from development to production
            
1)Docker Image:- Blueprint/template , Contains everything needed to run an app 
              :- Images are Immutable
              :- Stored in DockerHub or private registries for reuse
              :- Docker Images cannot be Updated
              :- Its the actual clone of our project

2)Docker Containers:- It is a running instance of a Docker Image (Object of Docker Image)
                   :- Isolated, Lightweight, Portable environment where apps run
                   :- Can be started, stopped, deleted
                   :- Docker install requirements like node and all even if it's not installed automatically, if its listed in requirements.txt

3)Vizualization:-
===========================
             build                      Run
Dockerfile-------------->Docker Images-------------->Docker Container

4)Base/Parent Image:- Starting point for building a Docker Image
                   :- can be official Images
                   :- Dockerfile inherits from a base image using FROM keyword

5)Docker Hub:-Official cloud based repository for container images 
            :-Provides pre-built images 
            :-Developers can push and pull images 
            eg:- node official image (type node/ubuntu/anything in docker hub search bar)

6)Docker Desktop:- GUI based tool to run Docker easily
                :- Comes with Docker Engine,CLI,and Kubernates support
                :- Helps in building,running,managing containers
                :- It has more features than docker hub

                           2)FROM <BaseImage>

                           3)COPY <source> <dest>  //making clone of our application
                           eg:- COPY . ./app  // .=>Copy from Current Directory , ./app=>paste in the directory(app) made in the image

                           5)EXPOSE <port no>
                           eg:- EXPOSE 4000
                           6)CMD ["node","index.js"]
                           eg:- CMD ["node","index.js"]
